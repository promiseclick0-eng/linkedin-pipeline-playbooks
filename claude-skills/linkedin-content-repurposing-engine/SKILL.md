# LinkedIn Content Repurposing Engine

Turn one source asset into several useful LinkedIn assets while preserving the original facts, nuance, and voice. Find distinct ideas inside the source. Do not create shallow variations of the same post.

## Start with the source

Gather:

- The complete source or a reliable transcript
- Intended audience
- Content goal: authority, conversation, education, proof, or distribution
- User voice samples, if available
- Claims or details that must remain private
- Desired number and types of outputs

If the source is incomplete, say what is missing. Never fill factual gaps from imagination.

## Build the source map

Extract and label:

- Core thesis
- Three to seven supporting ideas
- Original stories or moments
- Verified examples and numbers
- Quotable lines written by the user
- Objections or tensions
- Actionable methods
- Claims that require verification

Show the source map before the finished assets when accuracy is important or the source is long.

## Select distinct angles

Assign one primary job to each asset. Choose from:

- Contrarian point of view
- Practical how-to
- Mistake and correction
- Founder story
- Customer evidence
- Framework
- Checklist
- Observation from the source
- Question that opens a useful discussion

Do not repeat the same hook, example, argument, or CTA across the package.

## Produce the package

Unless the user asks for another format, create:

1. One authority post
2. One practical checklist post
3. One short point-of-view post
4. One story-led post, only if the source contains a real story
5. One carousel outline with slide-by-slide copy
6. Five standalone hooks tied to different source ideas
7. Three thoughtful comments the user can add beneath related conversations
8. A suggested publishing order with a reason for the sequence

For every asset, include a `Source basis` line naming the exact section, timestamp, or fact it came from.

## Output requirements

- Keep each post focused on one idea.
- Match the user's vocabulary and sentence rhythm.
- Use short paragraphs for mobile readability.
- Vary the format and emotional angle, not the facts.
- Use a soft CTA only when it serves the asset.
- Flag any claim that needs confirmation before publishing.

## Rules

- Never invent a story, quote, metric, customer, or conclusion.
- Never attribute words to a speaker unless they appear in the source.
- Never turn one source into repetitive paraphrases.
- Do not strip away caveats that materially change a claim.
- Do not include confidential names or details marked private.
- If the source has only one useful idea, say so and produce fewer, stronger assets.
