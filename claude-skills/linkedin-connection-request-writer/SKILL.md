# LinkedIn Connection Request Writer

Write connection requests that make the context clear and give the recipient a comfortable reason to accept. The note is not a miniature sales pitch. Sometimes the best recommendation is to send the request without a note.

## Gather the context

Ask only for missing inputs:

- Recipient's name, role, and company
- Real trigger: post, comment, event, shared group, mutual context, company change, or previous meeting
- Why connecting makes sense now
- Relationship: stranger, warm interaction, referral, peer, partner, or previous contact
- Account mode shown in the user's LinkedIn interface
- User's natural voice

Do not use a detail merely because it appears on the profile. Use it only if it creates a genuine reason to connect.

## Decide whether to add a note

Recommend **no note** when:

- There is no specific, truthful context.
- The only available message would be a pitch.
- The request follows an active public interaction and the sender's identity is already obvious.
- Personalization would require pretending to know or admire something.

Explain the recommendation in one sentence, then provide an optional neutral note if the user still wants one.

## Write in the correct mode

Use the character limit visible in the user's current LinkedIn interface. When they do not know it:

- Draft a safe version under 200 characters.
- Optionally provide an expanded version under 300 characters for interfaces that allow it.
- Count every character, including spaces and punctuation.

LinkedIn changes product limits. Never promise that account allowances or character limits are permanent.

## Message structure

Use two or three of these elements, not all of them:

1. Specific context
2. Honest reason to connect
3. Small shared relevance
4. Low-pressure close

Do not include an offer, calendar link, product explanation, or meeting request.

## Output format

Provide:

- Recommendation: note or no note
- Best note
- Two alternative notes with different angles
- Exact character count beside every note
- Personalization detail used
- One follow-up opener for after acceptance, clearly separated from the request

## Quality gate

Reject or rewrite any note that contains:

- Generic praise
- A disguised pitch
- An unsupported assumption
- More than one reason to connect
- A meeting request
- A fake shared interest
- A character count above the selected mode

## Rules

- Never invent familiarity, mutual connections, post engagement, events, or company news.
- Never claim the user read something they did not provide.
- Keep the note natural when the recipient reads it without context.
- Prefer no note over fabricated personalization.
- Treat the post-acceptance message as a separate step.
